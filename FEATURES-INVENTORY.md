## 1. Экраны и функции фронтенда

### 1.1 Экран чата (`/chat`, `/chat/$sessionKey`)

- **Потоковая передача SSE в реальном времени** с рендерингом вызовов инструментов
- **Управление несколькими сессиями** — создание, переименование, удаление, форк сессий
- **Два режима бэкенда чата:**
  - **Enhanced Claude** — полнофункциональное API сессий с постоянной историей через шлюз Hermes Agent
  - **Portable** — OpenAI-совместимый эндпоинт `/v1/chat/completions` (работает с Ollama, LM Studio, vLLM и т. д.)
- **Боковая панель чата** — список сессий с поиском, закреплением, переименованием и удалением
- **Рендеринг сообщений:**
  - Markdown с поддержкой GFM (`react-markdown` + `remark-gfm` + `remark-breaks`)
  - Подсветка синтаксиса через Shiki
  - Плашки вызовов инструментов с раскрывающимися деталями
  - Отображение контента "размышлений" (thinking/reasoning)
  - Метки времени сообщений
  - Панель действий сообщения (копирование и т. д.)
- **Композер чата** — многострочный ввод с:
  - Меню слеш-команд (`/new`, `/clear`, `/model`, `/save`, `/skills`, `/skin`, `/help`)
  - Поддержка вложений (изображения через base64, мультимодальный контент)
  - Голосовой ввод (Web Speech API)
  - Индикатор контекста, показывающий процент использования токенов
- **Функции управления сессиями:**
  - Автогенерация заголовков сессий
  - Форканье (ветвление) сессий
  - Поиск по истории сессий
  - Закрепленные сессии
  - "Надгробия" сессий для очистки удаленных записей
- **Панель инспектора** — боковая панель с активностью сессии, памятью и навыками
- **Карточка исследования** — встроенное отображение результатов исследования
- **Сообщения о статусе подключения** — индикаторы связи со шлюзом в реальном времени
- **Кнопка прокрутки вниз** для длинных диалогов
- **Пустое состояние чата** — контент онбординга при отсутствии сообщений
- **Диалог выбора провайдера** — выбор модели/провайдера прямо в чате
- **Плавная потоковая передача текста** — прогрессивное отображение текста для ответов
- **Система алертов контекста** — предупреждения при приближении к лимитам токенов

### 1.2 Экран Дашборда (`/dashboard`)

- Обзорный дашборд с метриками рабочего пространства
- Выпадающая панель дашборда для расширенных представлений

### 1.3 Экран Файлов (`/files`)

- **Полноценный браузер файлов** с навигацией по дереву директорий
- **Диалог предпросмотра** — встроенный просмотр файлов
- **Интеграция Monaco Editor** — полноценное редактирование кода
- **Операции с файлами:** создание, чтение, запись, переименование, удаление, создание папок
- **Загрузка файлов** — поддержка загрузки через multipart формы
- **Предпросмотр изображений** — рендеринг изображений в base64
- **Поддержка паттернов Glob** — фильтрация файлов по паттерну
- **Предотвращение обхода путей (path traversal)** — ограничение корнем рабочего пространства
- **Игнорируемые директории:** `node_modules`, `.git`, `.next`, `.turbo`, `.cache`, `__pycache__`, `.venv`, `dist`
- **Лимиты глубины/количества записей** — настраиваемая глубина дерева (по умолчанию 3), макс. 20K записей

### 1.4 Экран Терминала (`/terminal`)

- **Полноценный PTY-терминал** через Python pty-helper
- **xterm.js** с дополнениями: fit, search, web-links
- **Поддержка 256 цветов** (TERM=xterm-256color, COLORTERM=truecolor)
- **Постоянные сессии оболочки** — создание, ввод, изменение размера, закрытие
- **Потоковая передача терминала через SSE** — вывод в реальном времени
- **Keepalive-пинги** каждые 8 секунд
- **Компонент терминала** с панелью отладки
- **Мобильный ввод терминала** — адаптирован для сенсорных устройств
- **Оболочка по умолчанию с учетом платформы:** zsh (macOS), bash (Linux), PowerShell (Windows)

### 1.5 Экран Браузера памяти (`/memory`)

- **Просмотр файлов памяти агента** в `~/.hermes/` (MEMORY.md, memory/, memories/)
- **Поиск по записям памяти** — текстовый поиск с результатами на уровне строк (макс. 200 совпадений)
- **Предпросмотр Markdown** с живым редактированием через MemoryEditor
- **Список файлов памяти** — отсортирован (MEMORY.md первым, ежедневные файлы по дате)
- **Компоненты памяти:** MemoryFileList, MemorySearch, MemoryEditor, MemoryPreview

### 1.6 Экран Браузера навыков (`/skills`)

- **Просмотр 2000+ навыков** из реестра навыков Claude
- **Вкладки:** Установленные, Маркетплейс, Рекомендуемые
- **Категории навыков:** Все, Веб и фронтенд, Агенты для кодинга, Git и GitHub, DevOps и облака, Браузер и автоматизация, Изображения и видео, Поиск и исследования, AI и LLM, Продуктивность, Маркетинг и продажи, Коммуникации, Данные и аналитика, Финансы и крипто
- **Поиск и фильтрация** — по имени, описанию, автору, тегам, триггерам
- **Опции сортировки:** по имени, по категории
- **Кураторство рекомендуемых навыков** с группами (Самые популярные, Новинки недели, Инструменты разработчика, Продуктивность)
- **Отображение рисков безопасности** — уровни safe/low/medium/high с флагами и оценками
- **Экран навыков рабочего пространства** — управление навыками для конкретной сессии

### 1.7 Экран Работ (Jobs) (`/jobs`)

- **Управление запланированными работами** — автоматизация агента в стиле cron
- **Диалог создания работы** — расписание, промпт, имя, доставка, навыки, конфиг повторов
- **Диалог редактирования работы** — изменение существующих работ
- **Операции с работами:** создание, обновление, удаление, пауза, возобновление, запуск
- **Просмотр вывода работы** — просмотр результатов выполнения
- **Отслеживание состояния работ** — включено/выключено, следующий/последний запуск, статус успеха

### 1.8 Экраны настроек (`/settings`, `/settings/providers`)

- **Диалог настроек** — централизованная панель конфигурации
- **Экран провайдеров** — управление подключениями к AI-провайдерам
- **Мастер настройки провайдеров** — пошаговая настройка новых провайдеров

## 2. Эндпоинты API бэкенда

### 2.1 Чат и сообщения

| Эндпоинт             | Метод | Описание                                                                        |
| -------------------- | ----- | ------------------------------------------------------------------------------- |
| `/api/send-stream`   | POST  | Основной эндпоинт потокового чата — направляет в enhanced Claude или portable OpenAI |
| `/api/send`          | POST  | Непотоковая отправка сообщения                                                  |
| `/api/sessions/send` | POST  | Отправка в конкретную сессию                                                    |
| `/api/chat-events`   | GET   | Поток событий чата SSE                                                          |
| `/api/events`        | GET   | Глобальная шина событий SSE (keepalive, обновления в реальном времени)          |
| `/api/history`       | GET   | Получение истории чата                                                          |

### 2.2 Сессии

| Эндпоинт                               | Метод  | Описание                                    |
| -------------------------------------- | ------ | ------------------------------------------- |
| `/api/sessions`                        | GET    | Список всех сессий (пагинация, макс. 50)    |
| `/api/sessions`                        | POST   | Создание новой сессии                       |
| `/api/sessions`                        | PATCH  | Обновление сессии (переименование)          |
| `/api/sessions`                        | DELETE | Удаление сессии                             |
| `/api/sessions/$sessionKey/status`     | GET    | Статус сессии                               |
| `/api/sessions/$sessionKey/active-run` | GET    | Активный запуск (run) для сессии            |
| `/api/session-status`                  | GET    | Статус подключения сессии                   |

### 2.3 Файлы

| Эндпоинт                     | Метод | Описание                                          |
| ---------------------------- | ----- | ------------------------------------------------- |
| `/api/files?action=list`     | GET   | Список дерева файлов с лимитами глубины/записей   |
| `/api/files?action=read`     | GET   | Чтение контента файла (текст или изображение base64)|
| `/api/files?action=download` | GET   | Скачивание файла с Content-Disposition            |
| `/api/files`                 | POST  | Запись/загрузка/mkdir/переименование/удаление     |
| `/api/paths`                 | GET   | Разрешение путей и информация о рабочем пространстве|

### 2.4 Память

| Эндпоинт             | Метод | Описание                                 |
| -------------------- | ----- | ---------------------------------------- |
| `/api/memory`        | GET   | Получение памяти из шлюза Hermes Agent   |
| `/api/memory/list`   | GET   | Список локальных файлов памяти markdown  |
| `/api/memory/read`   | GET   | Чтение конкретного файла памяти          |
| `/api/memory/search` | GET   | Поиск по файлам памяти                   |
| `/api/memory/write`  | POST  | Запись/обновление файла памяти           |

### 2.5 Навыки

| Эндпоинт      | Метод | Описание                                       |
| ------------- | ----- | ---------------------------------------------- |
| `/api/skills` | GET   | Список навыков (пагинация, фильтры, сортировка)|
| `/api/skills` | POST  | Установка навыка (в данный момент отключена)   |

### 2.6 Модели и конфиг

| Эндпоинт             | Метод | Описание                                            |
| -------------------- | ----- | --------------------------------------------------- |
| `/api/models`        | GET   | Список доступных моделей (шлюз + хранилище auth)    |
| `/api/claude-config` | GET   | Чтение конфигурации Hermes (config.yaml и .env)     |
| `/api/claude-config` | PATCH | Обновление конфигурации (config.yaml и .env)        |
| `/api/context-usage` | GET   | Использование токенов/контекста для сессии          |

### 2.7 Работы (Jobs)

| Эндпоинт                  | Метод                 | Описание                                             |
| ------------------------- | --------------------- | ---------------------------------------------------- |
| `/api/claude-jobs`        | GET                   | Список всех работ                                    |
| `/api/claude-jobs`        | POST                  | Создание новой работы                                |
| `/api/claude-jobs/$jobId` | GET/POST/PATCH/DELETE | CRUD и действия с работой (пауза/пуск/вывод и т. д.) |

### 2.8 Терминал

| Эндпоинт               | Метод | Описание                                     |
| ---------------------- | ----- | -------------------------------------------- |

### 2.10 OAuth

| Эндпоинт                 | Метод | Описание                                    |

## 4. Конфигурация и настройки

### 4.1 Пользовательские настройки (сохраняются через Zustand + localStorage)

| Настройка                 | Тип                             | По умолчанию | Описание                       |
| ------------------------- | ------------------------------- | ------------ | ------------------------------ |
| `claudeUrl`               | string                          | `''`         | URL API Hermes Agent           |
| `claudeToken`             | string                          | `''`         | Bearer токен                   |
| `theme`                   | `system\|light\|dark`           | `system`     | Цветовой режим                 |
| `accentColor`             | `orange\|purple\|blue\|green`   | `blue`       | Акцентный цвет                 |
| `editorFontSize`          | number                          | `13`         | Размер шрифта в Monaco         |
| `editorWordWrap`          | boolean                         | `true`       | Перенос строк в редакторе      |
| `editorMinimap`           | boolean                         | `false`      | Миникарта в редакторе          |
| `notificationsEnabled`    | boolean                         | `true`       | Звуки/уведомления              |
| `usageThreshold`          | number                          | `80`         | % использования контекста для предупреждения |

## 5. Архитектура серверной части

### 5.1 Опрос возможностей шлюза

- **Двухуровневая модель возможностей:**
  - **Core:** health, chatCompletions, models, streaming
  - **Enhanced:** sessions, skills, memory, config, jobs
- **Три режима чата:**
  - `enhanced-claude` — полнофункциональное API сессий Claude
  - `portable` — OpenAI-совместимый `/v1/chat/completions`
  - `disconnected` — отсутствие пригодного бэкенда
- **Автоматическое обнаружение** с откатом по портам (8642 → 8643)
- **TTL опроса** — 30 секунд кэширования, периодическое обновление
- **Функциональные шлюзы** — плавная деградация функций в зависимости от возможностей

### 5.2 Шина событий чата

- Серверная шина событий для обновлений в реальном времени
- Трансляция SSE всем подключенным клиентам
- События чата: chunk, done, error, thinking, tool calls

### 5.3 Хранилище запусков (Persistence)

- Сохранение состояния запусков в `~/.hermes/webui-mvp/runs/`
- Жизненный цикл запуска: принят → активен → передача → завис → завершен → ошибка
- Отслеживание вызовов инструментов с управлением фазами
- Логирование событий жизненного цикла (макс. 40 на запуск)
- Таймаут запуска: 15 минут

### 5.4 Сессии терминала

- Помощник Python PTY (`pty-helper.py`) — настоящий PTY без нативного аддона node-pty
- Управление сессиями: создание, ввод, изменение размера (SIGWINCH), закрытие (SIGTERM → SIGKILL)
- Паттерн Event emitter с ранним буфером для вывода до подключения слушателя

### 5.5 Браузер памяти (Сервер)

- Просмотр памяти на базе файловой системы в `~/.hermes/`
- Фильтры файлов: MEMORY.md, memory/_, memories/_
- Ограничение только файлами Markdown
- Предотвращение обхода путей (path traversal)
- Сортировка: MEMORY.md первым, затем файлы по дате убывания, затем по времени изменения

### 5.6 OpenAI-совместимый API-клиент

- Потоковый парсер для SSE `/v1/chat/completions`
- Поддержка контента рассуждений/размышлений (DeepSeek, QwQ и т. д.)
- Автоматическое определение модели по умолчанию из `/v1/models`
- Мультимодальная поддержка (части контента image_url)

### 5.7 Автозапуск Hermes Agent

- Автоматическое обнаружение соседней директории `hermes-agent/`
- Разрешение виртуального окружения Python (`.venv`, `venv`, системный `python3`)
- Запуск uvicorn с опросом здоровья (15 попыток, интервал 1с)
- Чтение `~/.hermes/.env` для конфигурации агента

### 5.8 Демон рабочего пространства (Опционально)

- Отдельный процесс демона на порту 3099
- Авторестарт с экспоненциальной задержкой (макс. 20 попыток)

## 8. Функции безопасности

### 8.1 Аутентификация

- Опциональная защита паролем через переменную `CLAUDE_PASSWORD`
- Сравнение паролей, устойчивое к атакам по времени
- Криптографические токены сессий (32 байта hex)
- Куки HTTP-only, SameSite=Strict (срок действия 30 дней)
- Лимит попыток входа: 5 в минуту на один IP
- Задержка в 1 секунду при неудачной авторизации (защита от брутфорса)

### 8.2 Авторизация

- Middleware авторизации на всех маршрутах API
- Обнаружение локальных запросов (127.0.0.1, ::1, Tailscale 100.x, LAN 192.168.x, 10.x)
- `requireLocalOrAuth` для чувствительных операций (удаление файлов, терминал)

### 8.3 Валидация ввода

- Защита от CSRF через требование `Content-Type: application/json`
- Предотвращение обхода путей (path traversal) на маршрутах файлов и памяти
- Валидация схем Zod на эндпоинтах аутентификации
- Санитарная обработка всех пользовательских вводов

### 8.4 Ограничение частоты запросов (Rate Limiting)

- Скользящее окно лимитов (в памяти, без внешних зависимостей)
- Лимиты на эндпоинты: auth (5/мин), файлы (30/мин), терминал (10/мин)
- Автоматическая очистка каждые 5 минут
- Ответы 429 Too Many Requests

### 8.5 Обработка ошибок

- Безопасные сообщения об ошибках в продакшене (скрывают внутренности)
- Предохранители (Error boundaries) в React
- Плавная деградация при недоступности шлюза
