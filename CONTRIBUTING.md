# Участие в разработке Hermes Workspace

Спасибо за интерес к проекту! Вот с чего стоит начать.

## Быстрый старт

1. Сделайте форк репозитория и клонируйте его
2. Установите зависимости: `pnpm install`
3. Настройте окружение:
   ```bash
   cp .env.example .env
   # Отредактируйте .env — установите HERMES_API_URL (по умолчанию: http://127.0.0.1:8642)
   ```
4. Запустите API-сервер [Hermes Agent](https://github.com/NousResearch/hermes-agent)
5. Запустите сервер разработки: `pnpm dev`
6. Внесите изменения в отдельной ветке (feature branch)
7. Откройте PR в ветку `main`

## Разработка

```bash
# Установка зависимостей
pnpm install

# Сервер разработки (по умолчанию: localhost:3000)
pnpm dev

# Проверка типов
npx tsc --noEmit

# Линтинг
pnpm lint

# Сборка для продакшена
pnpm build
```

## Переменные окружения

Все параметры можно найти в `.env.example`. Основные из них:

- `HERMES_API_URL` — бэкенд шлюза Hermes Agent (по умолчанию: `http://127.0.0.1:8642`)
- `CLAUDE_PASSWORD` — опциональная защита веб-интерфейса паролем
- `CLAUDE_ALLOWED_HOSTS` — список хостов через запятую для доступа не через localhost

## Рекомендации

- **Один PR на одну функцию/исправление** — сохраняйте фокус
- **Тестируйте изменения** — убедитесь, что приложение собирается (`npx tsc --noEmit`) и запускается
- **Описывайте изменения** — четкий заголовок и описание PR
- **Никаких секретов** — никогда не коммитьте API-ключи, токены или пароли
- **Следуйте существующим паттернам** — придерживайтесь стиля кода, принятого в проекте
