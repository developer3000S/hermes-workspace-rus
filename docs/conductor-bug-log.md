# Журнал ошибок Conductor

## 1. Задания портативного Conductor никогда не выполнялись

- Симптом: миссии в переносимом режиме создавались как запланированные задания Hermes, но задания оставались в состоянии `scheduled` (запланировано) и никогда не запускались.
- Исправление: переносимый Conductor теперь использует существующий путь потоковой передачи сессии `/api/send-stream` вместо неработающего пути заданий.
- Проверка: дымовой тест переносимого API вернул SSE-события `started`, `chunk` и `done`, сборка прошла успешно.

## 2. Миссия на базе дашборда была запущена, но в UI отображалось `0 active`

- Симптом: страница проводника запускала миссию на базе дашборда, но панель активности оставалась в состоянии `0 active`, даже когда дашборд показывал активные сессии миссии.
- Исправление: фильтр сессий проводника теперь сопоставляет недавние сессии, связанные с миссией, по точному ключу и по тексту/описанию миссии, а не только по меткам `worker-*` / `conductor-*`.
- Проверка: после перезагрузки страницы проводника миссия показала `1 active`, и появилась карточка воркера.
