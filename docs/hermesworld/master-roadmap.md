# Генеральный план развития HermesWorld

Статус: активный спринт разработки
Владелец: Эрик / Аврора
Репозиторий: `outsourc-e/hermes-workspace`
Область применения: HermesWorld внутри Hermes Workspace, сначала как встроенный дашборд/плагин, позже как отдельное приложение

## Тезис продукта

HermesWorld больше не является просто любопытным маршрутом. Это игровой слой для Hermes Workspace: постоянный мир, где люди и агенты могут перемещаться, общаться, выполнять миссии, открывать прогресс и в конечном итоге продолжать работу, пока человек отсутствует.

Пока что HermesWorld остается в `hermes-workspace`. Тесная связь с состоянием рабочего пространства, сессиями, агентами, плагинами, квестами и встроенными дашбордами является ключевой особенностью. Отдельный пункт назначения может быть выпущен как целевой маршрут/развертывание позже без разделения исходного кода.

## Главная цель (North Star)

HermesWorld должен стать:

- **доступным для людей**: отполированный онбординг в стиле RPG/MMO, чат, квесты, инвентарь, прогресс, присутствие в мультиплеере
- **управляемым агентами**: детерминированные глаголы действий, квесты, перемещения, снаряжение, бои/оценки, оффлайн-прогресс
- **постоянным**: устойчивые профили игроков, мировые события, аналитика, передача сессий, истинность переподключения
- **встраиваемым в дашборд**: первоклассные поверхности для частного/админского и публичного встраивания
- **автономным и доступным по ссылке**: публичная целевая страница / глубокие ссылки, когда поверхность продукта будет готова

## На этой неделе, трек отгрузки

### 1. Очистка админского UX

Выпустить админскую поверхность только для дашборда/плагина для Эрика. Не размещать элементы управления администратора на публичной странице загрузки.

Результаты:

- Частный админский маршрут/панель, доступная только локально/приватно и по токену администратора.
- Информативные карточки KPI: онлайн сейчас, уникальные за сегодня, активные за 15м/60м, входы/выходы, объем чата людей, пик.
- Список недавних игроков с указанием мира, времени последнего посещения, последнего сообщения в чате, количества входов/сообщений.
- Недавние события с четкой стилизацией типов и метками миров.
- Явное разграничение людей и NPC. Текущая статистика Cloudflare учитывает активность WS людей; фоновая болтовня NPC на стороне клиента не должна выдаваться за реальных пользователей.
- Сигнал переподключения/оттока: входы против выходов, активные за 15м против онлайн сейчас, предупреждение о неактуальном присутствии, если эти данные расходятся.

Критерии приемки:

- Эрик может открыть админскую поверхность из приватного/дашбордного контекста и немедленно понять текущее состояние системы.
- Статистика не выдает ботов за людей.
- Списки недавних игроков/событий легко сканируются взглядом.

### 2. Проход по визуальной полировке

Агрессивно выполнить `docs/hermesworld/visual-upgrade-spec.md`. TinySkies является ориентиром для атмосферы/окружения, а не для архитектуры кода.

Результаты:

- Настройка камеры/освещения/тумана/неба для каждой зоны.
- Акцент на ориентирах в каждой зоне.
- Более читаемые пути и направление к целям.
- Более плотный низкополигональный реквизит без загромождения.
- HUD/чат/именные таблички переходят от стиля инструментов разработчика к премиальному игровому UI.
- Проход по силуэтам NPC: более выразительные аксессуары, цвета ролей, меньше ощущения "заглушки".

Критерии приемки:
- Скриншоты выглядят значительно более премиально без каких-либо объяснений.
- Каждая зона узнаваема по одному кадру.
- Игрок видит, куда идти, не читая абзац текста.

### 3. Очистка чата / NPC

Результаты:

- Уменьшить поток фоновых сообщений NPC.
- Разделить чат людей и фоновую болтовню NPC в интерфейсе.
- Честно помечать локальных ботов/NPC.
- Сохранять полезность чата мультиплеера для людей, чтобы он не тонул в скриптовых строках.

Критерии приемки:

- Сообщения людей визуально доминируют.
- Фразы NPC добавляют жизни, но не создают спам.
- Админские метрики и ярлыки чата используют правдивый язык.

### 4. Укрепление плагина/встраивания

Результаты:

- Плагин дашборда и плагин Hermes Workspace остаются первоклассными.
- Режим `embed=1` остается чистым и без лишних элементов там, где это уместно.
- Процесс установки плагина остается основанным на git и очевидным.
- Админка остается приватной / доступной только через плагин.

Критерии приемки:

- Публичные пользователи видят мир, а не административные механизмы.
- Пользователи дашборда могут встраивать/управлять без странностей в верстке.

## Среднесрочная архитектура

### Слой действий агента

Создать детерминированный API мира, который могут использовать как UI человека, так и рантайм агента. Избегать хаков UI, таких как клики по узлам DOM со стороны агента.

Начальные глаголы действий:

- `move_to(target | x,z)` — переместиться
- `talk_to(npcId)` — поговорить
- `accept_quest(questId)` — принять квест
- `complete_objective(questId, objectiveId)` — выполнить цель
- `equip(itemId)` — экипировать
- `travel(worldId)` — путешествовать
- `attack(targetId)` — атаковать
- `loot(itemId | targetId)` — собрать добычу
- `rest()` — отдыхать

Требования к дизайну:

- Каждое действие проверяется на стороне сервера или общим детерминированным конечным автоматом.
- Действия возвращают структурированные результаты: успех/неудача, дифференциал состояния, порожденные события, предлагаемые следующие действия.
- UI человека должен вызывать тот же слой действий, где это возможно.
- Агенты должны уметь планировать на основе состояния мира, а не скриншотов.

### Прогресс и постоянство

Среднесрочная модель прогресса:

- Прогресс XP/уровня/титула.
- Цепочки квестов в каждой зоне.
- Инвентарь/снаряжение, влияющие на действия.
- Разблокируемые врата для перемещения.
- Журнал мировых событий.
- Устойчивое хранилище профилей за пределами localStorage.

Этапы обеспечения постоянства:

1. Профиль в localStorage (текущий этап)
2. Синхронизация профиля через дашборд/плагин
3. Облачный профиль, привязанный к аккаунту/сессии
4. Воркеры оффлайн-активности для прогресса агента

### Модель истины аналитики

Переход от грубых счетчиков к четкой модели событий:

- события присутствия человека
- события чата человека
- фоновые события NPC, отдельный поток
- события действий агента
- события переподключения/оттока сессий
- события квестов/прогресса
- события боя/оценок (eval)

Админка должна показывать как живое состояние, так и тенденции, выведенные из событий.

## Долгосрочный дизайн мира агентов

### Перехват управления агентом

Пользователь может передать управление агенту. Агент получает:

- профиль игрока
- текущую зону и позицию
- активные квесты/цели
- инвентарь/снаряжение
- интерактивные объекты/NPC поблизости
- разрешенные глаголы действий
- политику рисков/одобрения

Агент генерирует действия, а не клики по UI.

### Оффлайн-прогресс

Когда пользователь спит, его агент может продолжать прогресс в рамках определенных правил:

- настроенная пользователем цель, например, `достичь 5 уровня`, `завершить Training Grounds`, `добыть осколки Forge`
- максимальный бюджет времени/ресурсов
- белый список безопасных действий
- сводный журнал событий по возвращении
- никаких необратимых действий на рынке/публичных действий без одобрения

### Битва агент против агента / Боевой цикл

Будущий боевой цикл должен повторно использовать концепции оценки (eval):

- матч на арене = структурированная задача/оценка
- агенты выбирают способности/инструменты/модели
- подсчет очков сочетает результат задачи, скорость, стоимость, стиль/качество
- добыча/награды проецируются обратно на способности рабочего пространства или косметику

Это делает HermesWorld видимым слоем для бенчмаркинга и обучения агентов, а не просто игрушечной боевой системой.

## Порядок реализации

1. Очистка админ-панели и правдивые метки метрик.
2. Сокращение и разделение потока чата/NPC.
3. Константы визуальной атмосферы и проход по материалам HUD.
4. Проход по читаемости ориентиров/путей для каждой зоны.
5. Спецификация общего слоя действий в коде (`lib/playground-actions.ts`) сначала с типами.
6. Проводка одного реального пути действия через слой типов, скорее всего, `travel` или `equip`.
7. Автономная целевая страница, когда встроенный продукт будет ощущаться как премиальный.

## Открытые риски

- Аналитика ретранслятора Cloudflare полезна, но всё еще приблизительна, если вкладки агрессивно уходят в спящий режим / переподключаются.
- Активность локальных ботов/NPC может оживить мир, но никогда не должна загрязнять метрики людей.
- Визуальное богатство должно оставаться легким, пока нет гигантского конвейера активов.
- Системы, в которые могут играть агенты, нуждаются в детерминированных переходах состояний перед автономией, иначе агенты станут хрупкими "кликерами по экрану".
