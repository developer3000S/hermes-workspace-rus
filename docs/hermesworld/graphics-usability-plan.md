# План развития графики и удобства использования HermesWorld

Статус: запуск сейчас
Ответственные: Эрик / Аврора
Стек: Hermes Workspace + React + Three.js / React Three Fiber

## Цель

Превратить HermesWorld из «многообещающего веб-демо» в «серьезный игровой мир» за счет:
- более сильной визуальной идентичности
- значительно улучшенного удобства использования (usability)
- более правдоподобных персонажей
- лучшего качества скриншотов и видеороликов
- четкого пути развития, остающегося в рамках веб-технологий

## Основное решение

**Не** менять движок сейчас.

Остаемся на:
- **Three.js**
- **React Three Fiber**
- текущей интеграции Hermes Workspace

Причины:
- важна возможность делиться ссылками в вебе
- важен трафик из X (Twitter)
- важен режим встраивания и дашбордов
- текущие барьеры — это арт-дирекшн, ассеты, рендеринг и UX, а не выбор движка

## Горькая правда о «реалистичных персонажах»

### Что реально достижимо сейчас
Мы можем прийти к:
- **стилизованному реализму / полуреализму / уровню премиальных легких MMO**
- выразительным силуэтам
- улучшенным лицам/волосам/одежде
- лучшей анимации
- более человеческим пропорциям
- улучшенным материалам и освещению

### Что нереально сейчас
Не в этом спринте:
- фотореалистичные люди уровня AAA Unreal
- полностью кастомная система главных героев с нуля
- сотни уникальных высококачественных персонажей

## Рекомендуемое направление персонажей

Целевой вид:
- **полуреалистичные персонажи фэнтези-RPG**
- более приземленные, чем в Roblox или низкополигональные фигурки
- менее пугающие («зловещая долина»), чем поспешный фотореализм
- читаемые на игровом расстоянии
- вписывающиеся в бюджеты производительности браузера

## Пайплайн персонажей (запуск сейчас)

### Лучший первый путь
Использовать проверенный источник персонажей вместо создания их из примитивов.

Рекомендуемый порядок:
1. **Ready Player Me** или аналогичный источник аватаров для быстрого получения правдоподобных баз людей
2. **Mixamo** для клипов анимации
3. Экспорт в **GLB**
4. Оптимизация материалов / размеров текстур для браузера
5. Адаптация гардероба/цветов под визуальный язык HermesWorld

### Почему именно этот путь
- быстрое получение правдоподобных людей
- работа с браузерным GLB-пайплайном
- вопрос анимации решается быстрее
- позволяет сосредоточиться на мире и UX, а не на создании персонажей с нуля

### Первый набор необходимых персонажей
- базовый мужской персонаж игрока
- базовый женский персонаж игрока
- ученый / NPC-оракул
- кузнец / NPC кузницы
- стражник / NPC-рыцарь
- торговец / NPC-горожанин

**Не** начинайте с огромного разнообразия.
Сначала создайте 4-6 отличных архетипов.

## Пайплайн анимации

### Первый пакет анимаций
Необходимы:
- покой (idle)
- ходьба
- бег
- разговор / жестикуляция
- осмотр / использование
- празднование / эмоция
- сидение или коленопреклонение (если легко реализовать)

### Цели поведения персонажей
- никаких застывших NPC-статуй
- анимация покоя должна создавать ощущение жизни
- движение должно четко считываться на расстоянии
- разговор должен казаться осознанным даже до реализации синхронизации губ (lip sync)

## Приоритеты улучшения окружения

## 1. Проработка достопримечательностей
Каждая зона должна иметь узнаваемый силуэт.

Немедленные цели:
- Центральный монумент Агоры / обелиск / алтарь сигилов
- Башня Оракула / кольцевая структура
- Печь кузницы / дымоход / источник тепла
- Древо памяти в Роще / кристаллическое древо / корни архива
- Кольцо арены / баннеры / ворота
- Лагерь тренировочных площадок / ворота / обучающая святыня

## 2. Проработка земли и путей
Исправить текущее ощущение «объектов на плоскости».

Необходимо:
- более выразительные каменные дорожки
- границы путей
- изменения высоты (рельеф)
- лестницы / склоны / края платформ
- сгруппированная растительность вместо случайного разброса

## 3. Группировка пропсов (объектов)
Перейти от общих разбросанных объектов к авторским композициям.

Необходимо:
- рыночные прилавки, которые действительно складываются в сцены
- группы скамеек / факелов / ящиков / бочек
- декоративные баннеры и указатели
- повторяющиеся наборы пропсов для каждой зоны

## 4. Освещение / атмосфера
Самое эффективное визуальное улучшение.

Необходимо:
- более сильное направление основного света (key light)
- туман / атмосферная дымка вдалеке
- теплые круги света от костров
- более холодный баланс теней
- мягкое свечение (bloom) / постобработка
- скайбокс (небо), поддерживающий мифическое настроение

## Приоритеты улучшения удобства использования (Usability)

## 1. Читаемость потока целей
Пользователь всегда должен знать:
- где он находится
- что делать дальше
- с чем можно взаимодействовать
- что делают его агенты

### Немедленные изменения
- более четкий виджет задач квеста
- более понятный язык путевых точек / маркеров
- улучшенная подсветка при наведении/взаимодействии
- более заметные подписи NPC

## 2. Очистка интерфейса (HUD)
Текущий HUD должен больше походить на игру и меньше на смесь слоев прототипа.

Необходимо:
- единый визуальный язык материалов HUD
- более четкая нижняя панель действий
- более аккуратная рамка мини-карты
- более чистая карточка характеристик игрока
- меньше визуального шума от остатков отладки/админки

## 3. UX чата / социальных функций
Чат не должен доминировать на сцене.

Необходимо:
- меньшая по размеру и более чистая панель чата
- лучший контраст и иерархия сообщений
- более удобное сворачивание/минимизация
- меньше перекрытий с игровым пространством

## 4. Подсказки ввода / взаимодействия
Нужна немедленная обратная связь:
- оформление приглашения к взаимодействию
- уверенность в цели клика
- обратная связь по пути к цели
- более четкое состояние выбранного NPC / объекта

## План производительности

Все это должно оставаться безопасным для работы в браузере.

### Правила
- отдавать приоритет **инстансингу** (instancing) для повторяющихся пропсов
- уменьшить количество уникальных материалов
- сжимать текстуры
- агрессивно ограничивать разрешение текстур
- использовать LOD (уровни детализации), где необходимо
- сдерживать постобработку
- проверять количество вызовов отрисовки (draw calls) после каждого этапа работы над артом

### Мышление в рамках бюджета
Для каждой основной сцены спрашивать:
- сколько персонажей на экране?
- сколько уникальных материалов?
- сколько динамических источников света?
- сколько эффектов прозрачности?

## Точные первые шаги реализации

### Шаг 1
Заменить текущие примитивные заглушки персонажей на импортированные гуманоидные модели GLB.

### Шаг 2
Добавить контроллер анимации для:
- покоя (idle)
- ходьбы
- жестикуляции/разговора

### Шаг 3
Перестроить центр Агоры, используя:
- более выразительный монумент
- радиальное мощение
- авторские группы пропсов
- осмысленное размещение NPC

### Шаг 4
Отшлифовать камеру и UI для новой сцены.

### Шаг 5
Сделать скриншоты/видео и настроить слабые места.

## Критерии успеха

Мы поймем, что это работает, когда:
- скриншот выглядит как реальная игровая сцена, а не прототип
- персонажи выглядят как люди, а не как заглушки
- цель и взаимодействия очевидны
- страница достойна публикации в X
- браузер по-прежнему работает плавно
