# Чек-лист по реалистичности Агоры

Статус: активный первый этап реализации
Владелец: Eric / Aurora

## Цель
Превратить Агору в первую зону, которая ощущается как настоящая игровая сцена, а не многообещающий прототип.

## Фаза 1 — Структура сцены
- [ ] изолировать логику сцены Агоры из гигантского файла `playground-world-3d.tsx`
- [ ] определить текущую модель игрока / путь рендеринга (render path)
- [ ] определить текущий путь рендеринга NPC
- [ ] определить места монтирования `PlayerCharacter` и `NpcCharacter`

## Фаза 2 — Персонажи
- [x] создать основу конфигурации архетипов персонажей
- [x] создать границы компонента `PlayerCharacter`
- [x] создать границы компонента `NpcCharacter`
- [ ] заменить один плейсхолдер игрока на `PlayerCharacter`
- [ ] заменить один плейсхолдер стража/оракула на `NpcCharacter`
- [ ] подключить поведение меток (labels) и выбора к новым компонентам персонажей

## Фаза 3 — Композиция Агоры
- [ ] усилить силуэт центрального монумента
- [ ] улучшить читаемость радиального мощения камнем / круглой площади
- [ ] более продуманно сгруппировать скамейки / лавки / факелы
- [ ] разместить NPC в заданных группах для общения
- [ ] удалить все явно "игрушечные" плейсхолдеры, блокирующие пространство

## Фаза 4 — Освещение и атмосфера
- [ ] улучшить направление основного света (key light)
- [ ] добавить более выраженные пулы теплого света от огня
- [ ] добавить контролируемый туман / атмосферу на расстоянии
- [ ] настроить блум/постобработку так, чтобы сцена выглядела насыщенной, а не размытой

## Фаза 5 — HUD и читаемость
- [ ] доработать панель целей (objective panel)
- [ ] уменьшить визуальный шум миникарты
- [ ] улучшить читаемость меток NPC
- [ ] улучшить подсказки для взаимодействия (interaction prompts)
- [ ] сделать нижнюю панель действий более похожей на игровой UI, а не на прототип

## Критерии успеха
- [ ] персонаж игрока выглядит как правдоподобный силуэт человека
- [ ] как минимум 3 NPC выглядят правдоподобно и отличаются друг от друга
- [ ] скриншот Агоры можно опубликовать в X (Twitter) без оправданий за качество
- [ ] поток целей (objective flow) понятен с первого взгляда
- [ ] никакие элементы локального интерфейса рабочего пространства (workspace chrome) не просачиваются в публичную/игровую часть
