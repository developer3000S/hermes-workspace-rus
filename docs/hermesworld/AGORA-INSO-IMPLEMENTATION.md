# План реализации центра Агоры — Справочник Inso

Справочник: `docs/hermesworld/reference-images/agora-center-inso-reference.jpeg`

## Что доказывает справочник

Цель — не просто модели с большим количеством полигонов. Это скомпонованный игровой экран:

- изометрическая камера / камера от третьего лица, направленная вниз на оживленную площадь
- круглая каменная ступица с центральным монументом/обелиском
- читаемые метки и группы NPC
- теплый свет факелов и синие акценты на крышах
- богатая детализация грунта: трава, цветы, швы дорожек, рыночные прилавки, скамейки
- островки HUD вместо сплошных плит:
  - статус игрока вверху слева
  - цель вверху по центру
  - миникарта вверху справа
  - чат внизу слева
  - панель действий внизу по центру
  - быстрые инструменты по правому краю
- плотность окружения по краям, свободный центр для прогулок

## Цель для HermesWorld v0.3

Создать сцену Агоры, которая ощущается как в справочнике, оставаясь при этом нативной для браузера:

1. Заменить пустую/плоскую ступицу на композицию круглой площади.
2. Добавить центральный ориентир/монумент, который служит точкой спавна.
3. Разместить NPC по кольцу, а не случайным образом.
4. Добавить рыночные прилавки, скамейки, факелы, листву и силуэты ворот портала.
5. Сделать визуальный стиль миникарты и HUD целей соответствующим игровому стилю.
6. Сохранить раздельный HUD для мобильных устройств, а не сжатую десктопную версию.

## Срезы реализации

### Срез 1 — Макет/Композиция

Файлы, которые, скорее всего, будут изменены:
- `src/screens/playground/components/playground-world-3d.tsx`
- `src/screens/playground/components/playground-environment.tsx`
- `src/screens/playground/components/playground-minimap.tsx`

Задачи:
- Добавить компонент `AgoraCommons`.
- Построить круглую площадь с каменными кольцами/плиткой.
- Добавить центральный обелиск/монумент.
- Переместить NPC в именованные точки привязки по кольцу.
- Оставить центр свободным для передвижения.

Критерии приемки:
- Скриншот десктопа читается как хаб Агоры при отдалении камеры.
- NPC не нагромождены визуально друг на друга.

### Срез 2 — Реализм освещения/материалов

Задачи:
- теплые огни факелов/фонарей вокруг кольца
- синие/голубые акценты на крышах/порталах
- контактные тени, фоновое затенение (ambient occlusion) / постобработка, если это не вредит производительности
- процедурный камень с текстурным видом через повторяющуюся геометрию/вариации материалов

Критерии приемки:
- скриншот имеет глубину, точку фокуса и контраст теплого/холодного.

### Срез 3 — Плотность реквизита

Задачи:
- прилавки с цветными навесами
- скамейки
- бочки/ящики/горшки
- группы цветов/травы
- камни по краям дорожек
- небольшая будка/ворота портала

Критерии приемки:
- края кажутся живыми, в то время как центр остается игровым.

### Срез 4 — Читаемость HUD/игры

Задачи:
- компактная карточка статуса вверху слева
- компактная карточка цели вверху по центру
- карточка миникарты вверху справа, стилизованная под справочник
- панель действий внизу по центру
- свернутый/компактный чат внизу слева

Критерии приемки:
- отсутствие загромождения сверху, четкая цель, карта, статус.

### Срез 5 — Цикл обновления активов

Задачи:
- генерация/уточнение арта зон, портретов NPC, иконок предметов, сигилов
- размещение активов в UI
- скриншот
- визуальный обзор
- доработка

Критерии приемки:
- сравнение скриншота браузера со справочником показывает видимое сближение.

## Распределение задач в рое (Swarm)

- Направление мобильного UX: HUD и ограничения отсутствия прокрутки.
- Направление геймплея/мира: привязки Агоры, композиция NPC, компонент площади.
- Направление арта: сгенерированные активы, справочники материалов, набор иконок UI.
- Направление рецензирования: производительность, размер бандла, безопасность секретов клиента.

## Жесткие ограничения

- Никаких секретов призов на стороне клиента.
- Никаких гигантских PR.
- Сохранять сгенерированные активы оптимизированными.
- Сохранять архитектуру хостинга движка.
- Мобильные устройства должны тестироваться отдельно.
