# HermesWorld — Библия систем Agentic WoW / Rohan

Последнее обновление: 2026-05-06

## Путеводная звезда

HermesWorld должна стать первой Agentic WoW: настоящим миром в стиле MMO, где люди и ИИ-агенты формируют группы (пати), гильдии, классы, билды, соперничество, рейды, события и участвуют в охоте за призами.

Референсы:
- World of Warcraft: рейды, роли, классы, подземелья, социальная идентичность, долгосрочная прогрессия
- Rohan / PlayRohan: расы, деревья навыков, войны гильдий, темп войн по выходным, PvP-идентичность
- Roblox: платформа для авторов, пользовательские комнаты/мини-игры, социальная игра
- Hermes/OpenClaw: агенты как нативные акторы, воркфлоу как квесты, модели как персонажи/инструменты

## Основная концепция

Основатель-одиночка плюс рой агентов строят ИИ-нативную MMO-компанию.
Игроки не просто используют агентов. Они привносят агентов в мир.

## Роли/классы игроков

Первоначальные классы должны быть простыми для понимания и легко соотноситься с поведением агентов.

### Классы для людей

- **Priest / Healer (Жрец / Лекарь)**: поддержка, баффы, восстановление, спасение, выживаемость группы
- **Guardian / Tank (Страж / Танк)**: поглощает урон, ставит щиты, защищает цели группы/гильдии
- **Mage / Promptcaster (Маг / Промпткастер)**: дистанционный взрывной урон, стихийные/логические заклинания, контроль толпы (crowd control)
- **Rogue / Scout (Разбойник / Разведчик)**: скрытность, обнаружение, подсказки по пасхалкам, быстрое перемещение
- **Engineer / Builder (Инженер / Строитель)**: создает инструменты, развертывает агентов, улучшает залы гильдий
- **Oracle / Analyst (Оракул / Аналитик)**: предсказание, понимание рынка/лора, планирование квестов
- **Bard / Social (Бард / Социальный класс)**: поднимает боевой дух, коммуникация гильдии, бонусы к репутации

### Специализации для агентов

- **Scout Agent (Агент-разведчик)**: исследует зоны, находит лор, составляет карты маршрутов
- **Scribe Agent (Агент-писец)**: записывает квесты, пишет заметки гильдии, резюмирует рейды
- **Builder Agent (Агент-строитель)**: создает инструменты, строит промпты, генерирует ассеты
- **Trader Agent (Агент-торговец)**: анализ рынка/призов/экономики
- **Combat Agent (Боевой агент)**: участвует в дуэлях/аренах/оценках (evals)
- **Healer Agent (Агент-лекарь)**: поддерживает действия группы, управляет восстановлением/ресурсами

## Модель дерева навыков

Заимствование глубины классов в стиле Rohan без излишнего усложнения на ранних этапах.

У каждого класса есть:
- 3 ветки
- 5 тиров (уровней) в каждой ветке
- активные навыки + пассивные модификаторы
- разблокировка через XP, квесты, достижения гильдии, сигилы

Пример: Priest
- Restoration (Восстановление): лечение, очищение, воскрешение, святилище
- Blessing (Благословение): баффы группы, прирост XP, щит, ускорение
- Oracle (Оракул): раскрытие подсказки, пророчество, сканирование античитом, понимание лора

Пример: Engineer
- Crafting (Крафт): создание предметов, модификации, ремонт
- Automation (Автоматизация): развертывание агента, очередь задач, создание инструмента
- Fortress (Крепость): улучшения зала гильдии, защита, ловушки

## Прокачка / прогрессия

Слои прогрессии:
- уровень игрока
- уровень класса
- опыт (XP) навыков
- уровень агента-компаньона
- уровень гильдии
- коллекция сигилов
- редкость предметов / экипировка (gear)
- репутация в каждой зоне/фракции

Награды:
- характеристики (stats)
- навыки
- титулы
- косметика
- зоны
- разрешения гильдии
- действия агентов
- состояния допуска к редким призам (авторитарно на сервере)

## Снаряжение / предметы / улучшения (attachments)

Слоты предметов:
- оружие/инструмент
- броня/роба
- реликвия
- слот сигила
- модуль компаньона
- талисман знамени гильдии

Характеристики предметов:
- HP / MP / SP (ОЗ / ОМ / ОВ)
- атака / сила заклинаний / защита
- отзыв / память / контекст (recall / memory / context)
- крафт / автоматизация / обнаружение
- аура группы / бонус гильдии

Улучшения (attachments):
- камни/сигилы
- модули агентов
- модификаторы заклинаний
- улучшения крафта
- визуальная косметика

## Системы гильдий

Гильдии должны быть ключевым отличием.

Возможности:
- создание гильдии + знамя
- зал гильдии / комната
- чат гильдии
- роли в гильдии
- еженедельные цели гильдии
- общее хранилище
- доска квестов гильдии
- рейды гильдии
- битвы гильдий
- таблица лидеров и сезонные награды

## Темп войн гильдий

Запланированные события в стиле Rohan:
- Окно войн гильдий по выходным
- Контроль карты на основе целей
- захват обелисков/сигилов
- защита реликвии в зале гильдии
- окно подсчета очков по времени
- режим зрителя / публичная трансляция
- награды: титулы, косметика, опыт гильдии, статус в таблице лидеров

Важно: запланированные события создают привычку и драму внутри сообщества.

## Рейды/подземелья

Групповой контент в стиле WoW:
- сначала подземелья на 3-5 игроков
- позже рейды на 10+ игроков
- агенты-компаньоны могут заполнять роли
- у боссов есть механики, а не только ОЗ (HP)
- требования к ролям важны: лекарь/танк/урон/поддержка/разведчик

Идеи для первых рейдов:
- Сломанная модель (The Broken Model)
- Ошибка Оракула (The Oracle's Fault)
- Двигатель галлюцинаций (The Hallucination Engine)
- Хранилище мертвого контекста (The Dead Context Vault)
- Титан черного ящика (The Black Box Titan)

## Мини-игры

Добавление социальных/виральных мини-игр:
- боксерская арена
- игры гильдий в спортивном стиле
- гонки / маршруты для спидрана
- арена дуэлей агентов BenchLoop
- захват сигила (capture-the-sigil)
- соревнования по крафту
- викторины / испытания на знание лора

## Система баланса

Каждый патч должен сопровождаться списком изменений (patch notes).

Отслеживание:
- соотношение урона/лечения/ресурсов
- кулдауны
- распределение редкости предметов
- частота выбора классов
- винрейты по классам/гильдиям
- инфляция экономики
- сила действий агентов

Публикация списка изменений патча:
- баффы
- нерфы
- новые предметы
- новые квесты
- расписание событий
- известные проблемы

## Нарратив для поглощения (Acquisition narrative)

Первая компания по разработке видеоигр с ИИ-агентами.
Основатель-одиночка + рой агентов строят первую агент-нативную MMO.
Целевая амбиция: создание категории, а не просто демо-версии.
