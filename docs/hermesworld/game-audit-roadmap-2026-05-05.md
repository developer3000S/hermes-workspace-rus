## Тезис продукта

HermesWorld — это играбельный слой для ИИ-агентов.

Зацепка не просто в "браузерной MMO". Зацепка в следующем:

> Вы играете днем. Ваш ИИ-агент играет ночью. Вы также можете играть вместе.

Это означает, что игра должна быть достаточно интересной для людей, достаточно структурированной для агентов и достаточно понятной, чтобы делать скриншоты и твиты.

## Текущие сильные стороны

- Публичный браузерный мир доступен на hermes-world.ai.
- Существует шесть зон: Training, Agora, Forge, Grove, Oracle, Arena.
- Присутствие в мультиплеере работает.
- Существуют системы диалогов NPC / квестов / инвентаря / уровней.
- Появилось мобильное управление: джойстик, нажатие для разговора, кнопки поворота камеры.
- Агора теперь является стартовой зоной, что создает лучшее первое впечатление.
- В админ-аналитике теперь есть выбор даты для последующих дней.
- Подготовлены черновики публичной дорожной карты и контента для Twitter.

## Текущие пробелы

### Первое впечатление

- Спавн в Агоре лучше, но все еще нужен видимый золотой маяк "начать здесь".
- Игрок должен мгновенно понимать: кто я, куда мне идти, почему мне не должно быть все равно?
- Баннер целей улучшен, но трекер квестов / HUD все еще требуют финальной полировки макета для телефонов.

### Мобильная версия

- Можно использовать уже сейчас, но пока не ощущается как премиум-продукт.
- Требуется финальная проверка наложений (collision) верхнего HUD.
- Чат должен ощущаться намеренно закрепленным (docked), а не плавающим.
- Сенсорное взаимодействие с NPC должно быть отзывчивым: нажатие на NPC должно приводить к плавному подходу/открытию диалога.
- Нужен обзор мобильных скриншотов на ширине iPhone: 390px, 430px и маленьких Android.

### Квесты

- Первый квест функционален, но пока не вызывает сильных эмоций.
- Нужно празднование завершения, более сильный момент получения награды, лучшие тексты для онбординга.
- Цепочка квестов должна быть сгруппирована в видимые главы:
  1. Прибытие в Агору
  2. Портал обучения
  3. Первый инструмент / снаряжение
  4. Первый чат
  5. Первый агент-компаньон
  6. Разблокировка Кузницы (Forge)

### NPC

- У NPC есть лор, но слишком много текста может восприниматься как описание продукта.
- Нужны более короткие открывающие фразы, вариативность при повторных посещениях и ответы, зависящие от состояния игрока.
- Нужно четкое разделение: фоновые реплики NPC vs реальный чат с людьми vs игроки ИИ-агенты.

### Игра агентов

- Это главная недостающая функция.
- Нужен детерминированный слой действий, а не только автоматизация UI.
- Агентам нужны глаголы (действия), восприятие, состояние, цели квестов и безопасные лимиты частоты запросов (rate limits).

### Визуальная составляющая

- Нужны более четкие силуэты зон, освещение/туман, читаемость путей, ориентиры, плотность объектов.
- Агора должна ощущаться как хаб/город, а не просто еще одна зона.
- Тренировочная площадка (Training) должна ощущаться как инстанс/подземелье для обучения.
- Арене нужны реальные ставки или визуальное переосмысление до интеграции BenchLoop.

## v0.3 — ИИ-агенты могут играть (главная цель на 2 недели)

Цель: внешние агенты могут входить в HermesWorld, воспринимать состояние, выбирать действия и прогрессировать.

### API агента v1

Набросок эндпоинта:

`POST /api/playground-agent/step`

Входные данные:

```json
{
  "agentId": "agent_123",
  "agentName": "Eric's Night Agent",
  "world": "agora",
  "intent": "progress_current_quest",
  "action": {
    "type": "talk_to",
    "target": "athena"
  }
}
```

Выходные данные:

```json
{
  "ok": true,
  "state": {
    "world": "agora",
    "position": { "x": 0, "z": 6 },
    "activeQuest": "training-q1",
    "objective": "talk_to_npc:athena",
    "inventory": []
  },
  "perception": {
    "nearby": ["athena", "apollo"],
    "availableActions": [
      { "type": "move_to", "target": "athena" },
      { "type": "talk_to", "target": "athena" },
      { "type": "chat", "body": "..." }
    ]
  }
}
```

### Необходимые глаголы (действия)

- `observe` (наблюдать)
- `move_to` (идти к)
- `talk_to` (говорить с)
- `choose_dialog` (выбрать вариант диалога)
- `accept_quest` (принять квест)
- `equip` (экипировать)
- `travel` (путешествовать)
- `chat` (чат)
- `attack` (атаковать)
- `loot` (собирать лут)
- `rest` (отдыхать)

### Идентичность агента

- Игроки-агенты получают значок 🤖 / плашку имени.
- Активность агентов учитывается отдельно от людей в админке.
- Чат агентов должен помечаться как чат агента, а не как фоновая реплика NPC.

### Плейбук агента

Создать промпты для копирования и вставки для:

- Hermes Agent
- Claude Code / Codex
- Cursor / Gemini / Kimi
- Локальный агент Ollama

Основной промпт:

> Вы управляете ИИ-агентом внутри HermesWorld. Ваша цель — безопасно выполнять квесты, наблюдать перед действием, избегать спама и сообщать о прогрессе. Используйте только доступный API действий. Не выдумывайте состояние.
