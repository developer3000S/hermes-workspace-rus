# Hermes Playground 🌐

> MMO для агентов. Браузерный 3D-мир, где вы можете ходить, общаться с NPC-агентами Hermes, выполнять квесты, повышать уровень и встречать других разработчиков. Создано для хакатона Nous Research × Kimi 2026.

## Питч

Документация — это скучно. Агенты — это абстрактно. Сообществам нужно общее пространство.

Поэтому **Hermes превращает онбординг в многопользовательский мир RPG**. Вы не просто читаете об агенте Hermes — вы *играете* в него. Пять миров, шесть зданий, в которые можно войти, город, полный NPC, объясняющих концепции памяти, инструментов и маршрутизации через квесты, а также многопользовательский режим присутствия, чтобы другие разработчики могли гулять по той же Агоре, что и вы.

## Попробовать

```bash
git clone https://github.com/outsourc-e/hermes-workspace
cd hermes-workspace
pnpm install
pnpm dev
# откройте http://localhost:3001/playground в двух вкладках браузера
```

Для реального многопользовательского режима между устройствами (без настройки, хостинговый хаб):

```bash
# Просто запустите pnpm dev. .env уже связывает VITE_PLAYGROUND_WS_URL с
# хабом на Cloudflare Worker + Durable Object по адресу:
#   wss://hermes-playground-ws.myaurora-agi.workers.dev/playground
# Откройте /playground на двух устройствах в разных сетях — они встретятся там.
pnpm dev
```

Хотите свой собственный хаб?

```bash
cd playground-ws-worker
pnpm install
pnpm wrangler login
pnpm deploy   # → wss://hermes-playground-ws.<ваш-субдомен>.workers.dev
# Затем установите VITE_PLAYGROUND_WS_URL + VITE_PLAYGROUND_STATS_URL в .env.production.
```

Локальный запуск (без облака):

```bash
# терминал A
pnpm playground:ws            # ws://localhost:8787
# терминал B
VITE_PLAYGROUND_WS_URL=ws://localhost:8787 pnpm dev
```

## Демо-сценарий (60 секунд)

1. Попадаете на титульный экран, вводите имя разработчика, настраиваете аватар, затем входите в Training Grounds (Тренировочные залы).
2. Подходите к Афине, принимаете Сигил Гермеса, затем открываете набор инструментов и надеваете Тренировочный клинок + Плащ новичка.
3. Отправляете одно локальное сообщение в чат, затем посещаете Подиум архива, чтобы изучить документацию, память и повторный вызов итераций.
4. Следуете по трекеру квестов к Forge Gate (Врата кузницы), просите Афину или Пана что-то создать и запускаете празднование завершения обучения.
5. Проходите через открытые Врата кузницы, видите короткую заставку "Генерация мира...", затем прибываете в Кузницу с живым эмбиент-звуком.
6. Атакуете мошенническую модель с помощью Strike / Dash / Bolt и кратко демонстрируете пульсацию при низком HP, если позволите ей нанести ответный удар.
7. Открываете вторую вкладку или устройство, чтобы показать присутствие других игроков, разработчиков поблизости, пинг именной таблички и живой чат.

## Подача на хакатон

Hermes Playground превращает онбординг агентов в цикл социальной RPG. Вместо чтения стены документации разработчики гуляют по общему миру, встречают NPC в тематике Hermes, изучают механику движений, снаряжение, чат, память и ритуалы сборки, а затем проходят через Врата кузницы в живое многопользовательское царство создателей. Это представляет Hermes Workspace как место, где вы живете, а не просто инструмент, который вы открываете.

### Сценарий демо на 30-60 секунд

1. "Это Hermes Playground, наша многопользовательская онбординг-RPG для хакатона Nous Research × Kimi."
2. "Новый разработчик начинает в Training Grounds, изучает пятиэтапный цикл под руководством Афины, Ириды и Пана."
3. "Трекер квестов, журнал, снаряжение, чат, документация и память — всё это соответствует реальным привычкам разработчика Hermes."
4. "Когда завершается последний шаг обучения, открываются Врата кузницы, и мы генерируем вступление в мир через маршрут NPC."
5. "Теперь мы в Кузнице, где промпты становятся инструментами, сражения — бенчмарками, а другие разработчики могут встретить вас в живой зоне."

## Архитектура

```
/playground (маршрут)
├── playground-screen.tsx           оркестратор + привязка HUD
├── playground-world-3d.tsx         сцена R3F, NPC/Боты/Удаленные игроки, интерьеры
├── playground-environment.tsx      многоразовые примитивы декораций/ориентиров
├── playground-hud.tsx              сферы характеристик (в стиле RuneScape)
├── playground-sidepanel.tsx        вкладки правой панели (инв/навыки/квесты/миры/настройки)
├── playground-actionbar.tsx        панель навыков
├── playground-chat.tsx             док чата
├── playground-dialog.tsx           разветвленные карточки диалогов NPC
├── playground-journal.tsx          журнал квестов
├── playground-map.tsx              модальное окно полноэкранной карты мира
├── playground-minimap.tsx          радар
└── hooks/
    └── use-playground-multiplayer.ts   транспорт BroadcastChannel + WebSocket
scripts/playground-ws.mjs           крошечный WS-ретранслятор (запуск через `pnpm playground:ws`)
```

Стек: TanStack Start + React Three Fiber + Drei + Three.js, ws (Node), BroadcastChannel API.

Никаких внешних 3D-активов — всё отрисовывается процедурно из примитивов, поэтому весь 3D-мир весит <250 КБ и запускается на любом ноутбуке.

## Стилизация > Фотореализм

Мы выбрали стилизованное инди-3D вместо фотореалистичного AAA. Обоснование:
- Браузер. Один разработчик. Ограниченное время хакатона.
- Кто угодно может присоединиться с любого устройства мгновенно.
- "Genshin-lite для агентов" воспринимается как осознанный выбор, а не недоработка.
- Будущая работа: аватары Ready Player Me + анимации Mixamo + r3f-postprocessing для следующего качественного скачка.

## Мультиплеер

В одном клиентском хуке параллельно работают два транспорта:

- **BroadcastChannel** — вкладки с одним источником находят друг друга мгновенно без сервера.
- **WebSocket** — крошечный ретранслятор без сохранения состояния (`scripts/playground-ws.mjs`) для разных устройств. Тот же формат данных.

Схема данных (отражает то, что будет использовать будущий сервер Colyseus / Durable Object):

```ts
type PresenceWire = { kind: 'presence'; id; name; color; world; interior; x; y; z; yaw; ts }
type ChatWire     = { kind: 'chat';     id; name; color; world; text; ts }
type LeaveWire    = { kind: 'leave';    id }
```

Варианты развертывания для WS-ретранслятора перечислены в `memory/goals/2026-05-03-playground-mmorpg/multiplayer-deploy.md` (Fly.io / Render / Railway).
