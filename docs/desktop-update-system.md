# Система обновления Hermes Workspace для десктопа

Эта ветка вводит контракт обновления, который должен использоваться при упаковке в DMG/EXE.

## Продукты

Hermes поставляется как два отдельно обновляемых продукта:

1. **Hermes Workspace**: оболочка пользовательского интерфейса и сервера.
2. **Hermes Agent**: локальная среда выполнения агента и шлюза.

Они не должны моделироваться как два удаленных репозитория в одном git-чекауте. Средство обновления Workspace обновляет Workspace. Средство обновления Agent обновляет установленный или встроенный в пакет Agent.

## API

- `GET /api/update/status`
  - возвращает состояние версии/установки/обновления для Workspace и Agent.
- `POST /api/update/workspace`
  - применяет обновление Workspace только тогда, когда это безопасно.
- `POST /api/update/agent`
  - применяет обновление Agent только тогда, когда это безопасно.

## Типы установки

Текущая реализация определяет:

- `git`: разработка или чекаут из исходников.
- `docker`: запуск в контейнере, обновление не применяется внутри процесса.
- `desktop`: зарезервировано для интеграции автообновления DMG/EXE.
- `unknown`: невозможно безопасно обновить автоматически.

## Поведение Git/dev

Для установок через git:

- Обновления Workspace используют `origin/<branch>` и требуют чистого чекаута с возможностью перемотки вперед (fast-forward).
- Обновления Agent вызывают собственную команду агента `hermes update` и требуют чистого чекаута Agent.
- Состояния с незакоммиченными изменениями (dirty) или без возможности fast-forward блокируются и отображаются как требующие ручного вмешательства, а не как основной путь через команду копирования.

## Поведение десктопа для последующей реализации

Упакованное приложение должно устанавливать `HERMES_WORKSPACE_DESKTOP=1` и предоставлять мост для обновления десктопной версии, который:

1. Проверяет подписанный манифест обновления или GitHub Release.
2. Скачивает обновление приложения Workspace через автообновление Electron или аналог.
3. Отдельно обновляет встроенный пакет Hermes Agent.
4. Перезапускает Workspace и Agent после обновления.
5. Сохраняет примечания к релизу для показа на первом экране после обновления.

Пользовательский интерфейс уже ожидает статус обновления на уровне продукта и примечания к релизу, поэтому десктопный мост должен соответствовать тому же контракту `/api/update/*`.
