# Состояние UI, создаваемое агентом

Hermes Workspace может отображать опциональное структурированное состояние пользовательского интерфейса, передаваемое агентом, вместо того чтобы полагаться исключительно на эвристический вывод панелей из обычного текста чата.

## Зачем это нужно

Некоторые боковые панели вызывают больше доверия, когда агент явно указывает, что именно должно быть отображено.

Это позволяет сохранять оболочку легкой:

- агент решает, что заслуживает места на экране
- клиент отрисовывает это
- эвристический запасной вариант остается только для подстраховки

## Текущая сфера применения

Первый этап поддерживает состояние, создаваемое агентом, для:

- событий артефактов на стороне чата в Инспекторе

Если артефакт, созданный агентом, отсутствует, существующее поведение пользовательского интерфейса остается без изменений.

## Примечания

- Это намеренно сделано опциональным и обратно совместимым.
- Точная форма протокола может измениться в будущем.
- Оболочка должна отдавать приоритет состоянию, созданному агентом, если оно присутствует, и консервативно возвращаться к стандартному поведению в противном случае.
