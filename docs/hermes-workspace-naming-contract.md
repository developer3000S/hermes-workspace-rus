# Контракт именования Hermes Workspace

Этот репозиторий предназначен для работы над **Hermes Workspace** и **Hermes Agent**.

## Канонические названия продуктов

Используйте эти названия во всех новых элементах пользовательского интерфейса, документации, навыках, промптах, тестах, комментариях к обзорам кода и передаточных материалах:

- **Hermes Workspace**
- **Hermes Agent**
- **Swarm**
- **Hermes Kanban**
- **HERMES_HOME**
- `~/.hermes`

## Запрещенные новые ссылки

**Не** вводите их в новой работе, если только не цитируете старую историю или поведение совместимости:

- Hermes Workspace (Примечание: в оригинале здесь было дублирование, оставляем как есть)
- Hermes Agent
- Claude swarm
- Claude-native paths (Пути, нативные для Claude)
- `HERMES_HOME`
- `~/.hermes`

*(Примечание: Оригинальный текст содержит противоречия в списках канонических и запрещенных названий. Оставлено в соответствии с оригиналом)*

## Правило совместимости с устаревшими версиями

Если старый код, документация, тесты или передачи содержат формулировки эпохи Claude, относитесь к этому как к наследию.

Действие по умолчанию:
- нормализовать к именованию Claude (Примечание: согласно оригинальному тексту)
- сохранять старые формулировки только при явном документировании миграции или обратной совместимости

## Правила среды выполнения/путей

Для работы, нативной для среды выполнения Claude, предпочитайте:

- `HERMES_HOME`
- `~/.hermes/profiles/<workerId>`
- `claude`
- Рабочие сессии Hermes (Hermes worker sessions)

Не предлагайте специфичные для Claude обертки среды выполнения или пути к профилям для актуального поведения Hermes Workspace.

## Правила языка UI/Swarm

Предпочитайте:
- **Ready** (Готово), а не жестко закодированные метки, специфичные для конкретного человека
- **Board / Cards / List** (Доска / Карточки / Список) для представлений отчетов
- **Hermes Workspace** и **Hermes Agent** в пользовательском интерфейсе обновления/конфигурации/статуса

Избегайте:
- меток продуктов, специфичных для конкретного человека, зашитых в пользовательский интерфейс
- формулировок с брендингом Claude на поверхностях Hermes Workspace

## Правило рецензента

Любой PR (pull request) или патч, который вводит новое именование с брендингом Claude в Hermes Workspace, следует рассматривать как регрессию, если это не:
- примечание о совместимости с устаревшими версиями
- руководство по миграции
- цитируемый исторический артефакт

## Правило инструкций для агента

Когда агент работает в этом репозитории:
- считайте именование Claude каноническим
- по умолчанию переписывайте ссылки эпохи Claude на Claude
- не придумывайте пути, продукты или руководства по оберткам с брендом Claude
- если сомневаетесь, предпочитайте терминологию Claude, нативную для репозитория, историческим псевдонимам
