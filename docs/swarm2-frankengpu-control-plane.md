# Бриф плоскости управления Swarm2 FrankenGPU-7777

## Концепция продукта

Swarm2 должен восприниматься как раздел "Операции", усиленный агентами роя: Aurora является видимым узлом маршрутизации, воркеры — это операционные карточки, подключенные к этому узлу, а нижний чат маршрутизатора остается мозгом оркестрации. Топология должна прояснять координацию, а не заменять карточки.

## Стандартная структура главной страницы

1. Заголовок / управление режимами
   - Держать `/swarm2` отдельно от текущего `/swarm`.
   - Режим по умолчанию — `Control plane` (плоскость управления) / топология карточек.
   - Runtime / tmux — явный вторичный режим, никогда не являющийся основной поверхностью.

2. Основной хаб Aurora
   - Большая карточка оркестратора в верхней центральной части с пометкой Aurora / Orchestrator.
   - Показывает агрегированное операционное состояние: воркеры онлайн, количество участников в комнате/подключенных, состояние авторизации, выбранный воркер.
   - Содержит элемент управления для открытия нижнего чата маршрутизатора.
   - Визуально сильнее любой карточки воркера.

3. Слой соединений (Wiring layer)
   - Видимые соединительные линии SVG/CSS идут от Aurora вниз к полю воркеров.
   - Линии по умолчанию приглушены, подсвечиваются для участников комнаты и выбранного воркера.
   - Провода подчеркивают пути маршрутизации; они не становятся основной моделью взаимодействия.

4. Карточки узлов-воркеров
   - Карточки остаются основной операционной поверхностью.
   - Каждая карточка воркера должна продолжать показывать роль, состояние, текущая задача, последний полезный сигнал и прямые действия.
   - Карточки располагаются под Aurora или вокруг неё в центрированном поле узлов, а не на обычной странице с сеткой.
   - Прямые возможности остаются: фокус/выбор, добавление/удаление из комнаты, действия с задачами/маршрутизатором, действия с терминалом/рантаймом.

5. Нижний чат маршрутизатора
   - Остается снизу по центру и может быть закреплен.
   - Он представляет собой активный мозг оркестрации для ручной или широковещательной диспетчеризации.
   - Карточки воркеров и карточка Aurora открывают/фокусируют его, но он не заменяет карточки воркеров.

6. Runtime/tmux
   - Остается отдельным режимом.
   - Режим рантайма по-прежнему может использовать выбранного воркера или цели в комнате.
   - Панели терминала не должны доминировать на основной поверхности по умолчанию.

## Текущий этап реализации

- Заменить компактную горизонтальную полосу топологии в качестве основной визуальной метафоры на сцену плоскости управления.
- Добавить карточку Aurora, центрированную над карточками воркеров.
- Отрисовать видимый SVG соединений позади сцены воркеров с детерминированными линиями от хаба к каждому слоту карточки воркера.
- Сохранить `AgentCard` в целости, чтобы операционные детали и действия оставались реальными.
- Сохранить существующий `AttentionRail` в качестве вторичной боковой панели статуса, а не основной поверхности.

## Ограничения

- Никаких заглушек или текстов-заполнителей.
- Никаких чисто декоративных карточек, скрывающих операционное состояние.
- Избегать загромождения текущего Swarm: никаких плотных рядов несвязанных виджетов, никакой стены терминалов по умолчанию.
- Сохранять стабильность `/swarm`; изменять только `/swarm2` и общие компоненты, уже введенные для swarm2.

## Дальнейшие цели после этого этапа

- Измерять реальные позиции в DOM для точных кривых Безье вместо линий на основе слотов макета.
- Добавить анимацию активности маршрутов от Aurora к воркерам во время диспетчеризации и получения результатов.
- Позволить воркерам группироваться по ролям/линиям, сохраняя полный размер карточек.
- Выносить результаты очередей/диспетчеризации на хаб Aurora.
- Добавить визуальные пресеты комнат, не превращая поверхность в редактор графов.
