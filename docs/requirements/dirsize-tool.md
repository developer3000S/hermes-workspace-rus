# Анализ требований: Инструмент статистики размера директорий (dirsize-tool)

## 1. Исходное требование

Подсчет общего размера всех файлов в указанной директории с поддержкой человекочитаемых единиц измерения и структурированного вывода.

## 2. Сценарии использования

### Сценарий A: Временная статистика (легкий вариант)
- Пользователь хочет быстро узнать размер конкретной директории.
- Ожидаемое решение: достаточно команды `du -sh <path>`.

### Сценарий B: Инструмент для разработки/тестирования (структурированный вариант)
- Используется как инструмент проверки для тестов исправления @mention.
- Требуется предсказуемый структурированный вывод для автоматизированных утверждений (assertions).
- Требуются низкие побочные эффекты (только чтение) и быстрое выполнение.
- Требуется как вызываемый инструмент внутри экосистемы Hermes.

## 3. Сравнение трех вариантов

| Вариант | Сложность | Читаемость | Масштабируемость | Сценарий применения |
|------|--------|--------|---------|---------|
| A. `du -sh` | 0 (встроено в систему) | Плохая (каждый раз вводить строку параметров) | Нет | Временная статистика |
| B. Скрипт на Python | Низкая (~30-40 строк) | Хорошая (аккуратный CLI) | Высокая (можно добавить фильтрацию/сортировку) | Инструмент для разработки/тестирования |
| C. Инструмент Hermes MCP | Средняя (требуется регистрация и настройка) | Хорошая (вызов через цепочку инструментов) | Высокая | Глубокая интеграция |

**Рекомендуемый вариант: B (скрипт на Python)** — баланс легкости и масштабируемости, расположение в `scripts/dirsize.py`.

## 4. Граничные случаи и риски

| Сценарий | Влияние | Способ обработки |
|------|------|---------|
| Директории/файлы с недостаточными правами | Неполная статистика, генерация исключения | Добавить флаг `--ignore-permission-denied` для пропуска |
| Символические ссылки | Циклические ссылки могут привести к бесконечному обходу | `follow_symlinks=False` (не следовать по умолчанию) |
| Сверхбольшие директории (миллионы файлов) | `os.walk` работает медленно, но выполнимо | Замена на `os.scandir` может ускорить; добавить `--max-depth` |
| Разреженные файлы (sparse files) | `getsize` сообщает видимый размер (apparent size) | Разделение `--apparent-size` (по умолчанию) и `--disk-usage` |
| Именованные каналы (pipes) / файлы устройств | `getsize` возвращает 0 | Считать только `isfile()` |

## 5. Заключение

- Если это временная потребность: достаточно `du -sh`, разработка не требуется.
- Если это инструмент для тестирования: вариант со скриптом на Python оптимален (30-40 строк), поддержка вывода `--json` удовлетворяет потребностям автоматизированных утверждений.
- Принадлежность пути ожидает подтверждения от Carlo (глобальный инструментарий Hermes против `scripts/` проекта).
