# Участие в переводе интерфейса

Hermes Workspace в настоящее время использует легковесную карту переводов для строк интерфейса, которые были подготовлены для локализации.

## Файл переводов

Переводы находятся в:

```text
src/lib/i18n.ts
```

Важные части:

- `EN`: исходные английские ключи.
- `ZH`: переводы на упрощенный китайский.
- `RU`: переводы на русский язык.
- `LOCALES`: сопоставляет идентификаторы языков с картами переводов.
- `LOCALE_LABELS`: метки, отображаемые в селекторе языка.

## Добавление или улучшение переводов

1. Откройте `src/lib/i18n.ts`.
2. Найдите объект соответствующего языка (например, `RU` для русского).
3. Обновите значение справа от каждого ключа.

Пример:

```ts
const RU: LocaleTranslations = {
  'nav.dashboard': 'Панель',
  'nav.chat': 'Чат',
}
```

Оставляйте названия ключей без изменений. Редактируйте только переведенный текст.

## Добавление нового переводимого текста UI

Если вы нашли жестко закодированный английский текст в UI:

1. Добавьте новый ключ в `EN`.
2. Добавьте этот же ключ во все карты локалей (`ZH`, `RU` и т.д.).
3. Замените жестко закодированный текст в компоненте на `t('ваш.новыйКлюч')`.

Пример:

```ts
// src/lib/i18n.ts
const EN = {
  'common.retry': 'Retry',
} as const

const RU: LocaleTranslations = {
  'common.retry': 'Повторить',
}
```

Затем в компоненте:

```tsx
import { t } from '@/lib/i18n'

;<button>{t('common.retry')}</button>
```

## Локальное тестирование

Запустите:

```bash
pnpm exec vitest run src/lib/i18n.test.ts
pnpm build
```

Затем откройте "Settings" → "Language" (Настройки → Язык) и переключитесь на нужный язык.

## Текущие ограничения

Еще не все строки интерфейса перенесены в карту переводов. Если после смены языка текст остается на английском, это, скорее всего, означает, что в данном компоненте всё еще используется жестко закодированный текст, и его нужно сначала подключить к `t(...)`.
