# Навыки Swarm

Навыки — это переиспользуемые операционные знания, стоящие за режимом Swarm. Пресет роли определяет навыки, которые должен загрузить воркер; профиль и путь к навыкам делают их доступными во время выполнения (runtime).

Навык — это не просто "атмосфера". Это процедура: когда его использовать, какие команды запускать, какие файлы важны, какие ловушки существуют и как проверить результат.

## Встроенные навыки роя (Swarm Skills)

| Навык | Для чего использовать |
| --- | --- |
| `swarm-orchestrator` | Цикл оркестратора, диспетчеризация, обнаружение отклонений, повторные промпты, эскалация, маршрутизация миссий. |
| `swarm-worker-core` | Базовый контракт воркера: фазы, контрольные точки, состояние рантайма, блокировщики, передачи. |
| `swarm-review-learning-loop` | Фиксация извлеченных уроков после проверки, повторяющихся сбоев и улучшение навыков после выполнения задач. |
| `byte-verified-code-review` | Проверка диффов с байтовым подтверждением для чувствительных к именованию и сгенерированных файлов. |
| `swarm-bench-worker` | Лабораторная работа/бенчмарки для локальных моделей, эксперименты с рантаймом и логирование результатов. |
| `swarm-pr-worker` | Рабочий процесс с GitHub issues/PR, триаж, патчинг, подготовка PR, обратная связь по проверкам. |
| `swarm-ui-worker` | Канал реализации UI для поверхностей Hermes Workspace. |
| `swarm-dev-runtime` | Контракты рантайма, API бэкенда, жизненный цикл, исправность и настройка ремонта. |
| `swarm-memory` | Ожидания по работе с файловой памятью для воркеров и истории оркестратора. |
| `swarm-orchestration-loop` | Канонический цикл оркестрации/проверки для флотов постоянных воркеров. |

В некоторых установках навык байтовой проверки может называться иначе. Если точный навык отсутствует, используйте доступный навык проверки, который обеспечивает контроль байтов, проверку диффов, тесты, сборку, дымовые тесты и дисциплину вердиктов.

## Как навыки загружаются автоматически

Когда вы клонируете репозиторий и запускаете рабочее пространство, сессии воркеров загружают навыки из настроенного профиля агента Hermes. В релизной среде Eric профили указывают на общую директорию навыков, которая обычно доступна воркерам по пути:

```text
~/.ocplatform/workspace/skills/
```

Профиль воркера также может содержать локальные навыки профиля по пути:

```text
~/.hermes/profiles/<workerId>/skills/
```

Важное правило заключается в том, что рантайм воркера должен иметь возможность найти название навыка в своем профиле. UI может отображать навыки роли по умолчанию, но воркеру все равно нужны файлы навыков, доступные локально.

## Что дает каждый основной навык

### swarm-worker-core

Базовый контракт для каждого агента Hermes:

- понять задачу
- установить фазу
- выполнить следующий значимый шаг
- проверить
- поставить контрольную точку
- продолжить, эскалировать или чисто остановиться

Если у воркера есть только один навык, это должен быть именно он.

### swarm-orchestrator

Используется в канале оркестратора. Он отвечает за:

- декомпозицию миссий
- маршрутизацию на основе ролей
- обнаружение отклонений
- интерпретацию контрольных точек
- повторные промпты
- эскалацию
- приоритет каналов
- гигиену передач

Оркестратор не обязан быть лучшим кодером. Это воркер, который делает полезным каждого другого воркера.

### swarm-review-learning-loop

Используется после завершения работы и проверок для преобразования результатов в память или улучшения навыков. Это предотвращает повторные ошибки роя на одних и тех же граблях под видом "исследований".

### byte-verified-code-review

Используется для шлюзов проверки. Он заставляет предоставлять доказательства вместо "выглядит нормально":

- проверка точного диффа
- байтовая проверка хрупких именований или сгенерированного вывода
- запуск тестов/сборки/дымовых тестов
- вынесение вердикта
- точное документирование блокировщиков

### swarm-bench-worker

Используется Lab для экспериментов с локальными моделями и рантаймом:

- план бенчмарка
- контролируемый запуск
- фиксация результатов
- заметки о воспроизводимости
- порог эскалации

### swarm-pr-worker

Используется каналами PR/issues:

- сканирование issues
- оценка (scoring)
- воспроизведение
- дисциплина веток
- подготовка исправлений/тестов/PR
- обработка обратной связи по проверкам

### swarm-ui-worker

Используется создателями UI:

- инспекция маршрутов
- границы компонентов
- визуальное состояние
- дымовые тесты
- верификация сборки

### swarm-dev-runtime

Используется каналами Foundation/runtime:

- контракты API
- состояние профиля/рантайма
- проверки исправности
- ремонт жизненного цикла
- интеграция tmux/шлюза

## Добавление пользовательских навыков

Создайте директорию навыка с файлом `SKILL.md`:

```text
skills/my-skill/
  SKILL.md
  references/
  templates/
  scripts/
```

Рекомендуемые метаданные (frontmatter):

```yaml
---
name: my-skill
description: Одно предложение, объясняющее, когда воркер должен загрузить этот навык.
---
```

Рекомендуемые разделы:

```markdown
# My Skill

## Триггер
Когда использовать.

## Шаги
1. Точный первый шаг.
2. Точный второй шаг.
3. Верификация.

## Ловушки
- Известный режим сбоя.
- Точное действие для разблокировки.

## Контракт вывода
Что воркер должен вернуть.
```

## Добавление пользовательского навыка воркеру

1. Добавьте папку навыка в общую директорию навыков или в директорию `skills/` профиля воркера.
2. Добавьте название навыка в пресет роли или запись в списке воркеров.
3. Перезапустите или обновите сессию воркера, чтобы профиль перезагрузился.
4. Отправьте небольшую задачу, требующую этого навыка.
5. Проверьте, что в контрольной точке указан навык и приведены доказательства.

## Правила гигиены навыков

Навык следует обновлять, когда:

- команда устарела
- путь изменился
- предположение о настройке неверно
- воркер столкнулся с повторяющейся ошибкой, не описанной здесь
- изменился предпочтительный рабочий процесс пользователя
- существует лучший шаг верификации

Не создавайте заметку в памяти для процедуры, которая относится к навыку. Память хранит долговечные факты; навыки хранят повторяющиеся рабочие процессы.

## Чек-лист загрузки навыков

Прежде чем винить воркера:

- Существует ли навык с точным названием?
- Есть ли у профиля воркера доступ к директории навыков?
- Началась ли сессия после добавления навыка?
- Включает ли пресет роли этот навык?
- Требовала ли задача от воркера загрузить его?
- Содержит ли контрольная точка доказательства того, что процедура была соблюдена?

## Минимально жизнеспособный набор навыков для Swarm v1

Для полезного воркера:

```text
swarm-worker-core
```

Для полезного оркестратора:

```text
swarm-orchestrator
swarm-worker-core
swarm-review-learning-loop
```

Для полезного рецензента:

```text
swarm-worker-core
byte-verified-code-review
swarm-review-learning-loop
```

Все остальное — это специализация.
