## 🔴 Высокий приоритет (открывает возможность ночных запусков "Фабрики приложений")

### 1. Цикл итеративного улучшения (Iterative Refinement Loop)

**Что:** Проверка не останавливается на одном проходе tsc. Цикл: запуск tsc → ошибки? → отправка обратно агенту → исправление → повторный запуск. Максимум 3 итерации перед передачей на проверку человеку.
**Зачем:** Anthropic явно идентифицирует это как паттерн, который делает агентов надежными. Текущий однопроходный подход может приводить к скрытым ошибкам.
**Где:** `workspace-daemon/src/verification.ts` + `checkpoint-builder.ts`
**Источник паттерна:** Anthropic Skills Guide — дизайн-паттерн "Iterative Refinement"

### 2. Передача контекста между агентами (Agent Handoffs)

**Что:** Когда один агент заканчивает "волну" работы, он передает структурированный контекст (git diff, лог ошибок, что было создано, что пропущено) следующему агенту. Больше никаких запусков "вслепую".
**Зачем:** Сейчас агенты начинают каждую задачу с нуля. Передача контекста (handoffs) — это базовый элемент в OpenAI Agents SDK, обеспечивающий явную передачу управления с сохранением контекста. Это то, что делает ночные запуски последовательными.
**Где:** Новый `workspace-daemon/src/handoff.ts`, обновление интерфейсов адаптеров
**Источник паттерна:** OpenAI Agents SDK — примитив "Handoffs"

### 3. Специализированные роли агентов

**What:** Замена универсального адаптера Codex на агентов со специфическими ролями:

- **Исследователь (Researcher)** — читает кодовую базу, создает спецификацию/контекстный документ
- **Планировщик (Planner)** — берет спецификацию, создает декомпозицию задач с зависимостями
- **Строитель (Builder)** — выполняет задачи (Codex)
- **Валидатор (Validator)** — запускает tsc, тесты, проверяет diff
- **Деплоер (Deployer)** — операции с git, создание PR, уведомления
  **Зачем:** Сценарий "Фабрики приложений" запускает специализированные роли. Универсальные агенты упускают контекст предметной области.
  **Где:** `workspace-daemon/src/adapters/` — по одному файлу на роль
  **Источник паттерна:** Anthropic Skills — "Domain-specific intelligence" + паттерн App Factory

## 🟡 Средний приоритет

### 4. Параллельные проверки (tsc watcher во время работы агента)

**Что:** Запуск tsc в режиме watch параллельно с работой Codex, а не только после нее. Пометка ошибок в реальном времени без ожидания контрольной точки (checkpoint).
**Зачем:** OpenAI SDK запускает проверки (guardrails) параллельно с агентом — это позволяет выявлять проблемы, не блокируя основной поток.
**Где:** Новый процесс, запускаемый вместе с агентом в `agent-runner.ts`
**Источник паттерна:** OpenAI Agents SDK — примитив "Guardrails"

### 5. Откат при отклонении контрольной точки

**Что:** При отклонении контрольной точки — автоматический возврат к состоянию git до начала задачи, вместо того чтобы оставлять "грязный" код в дереве.
**Зачем:** Сейчас отклонение оставляет сломанный код, который наследует следующий агент.
**Где:** `workspace-daemon/src/git-ops.ts` — добавить метод `revertToCheckpoint()`

### 6. Контекстно-зависимый выбор инструментов

**Что:** Логика маршрутизации агента, которая выбирает разные инструменты в зависимости от размера файла, типа задачи и контекста. Большие рефакторинги → Codex. Маленькие точечные исправления → сессия Claude ACP. Задачи по исследованию → Claude с веб-поиском.
**Источник паттерна:** Anthropic Skills — паттерн "Context-aware tool selection"

## 🔵 Низкий приоритет (Enterprise / Масштабируемость)

### 7. Доступ агентов к истории сессий

**Что:** Передача контекста предыдущих запусков (что сработало, что не удалось, история git) агенту в начале каждой задачи. Сейчас агенты начинают работу "вслепую" даже при повторном запуске.
**Где:** Обновление адаптера `buildPrompt()`, чтобы включить историю запусков из SQLite

### 8. Прогрессивная загрузка навыков для промптов агента

**Что:** Системные промпты агента используют трехуровневое прогрессивное раскрытие Anthropic — минимальный заголовок загружен всегда, полные инструкции — только при срабатывании триггера, справочная документация — по запросу.
**Зачем:** Позволяет экономить контекст при параллельном запуске множества агентов.
**Источник паттерна:** Anthropic Skills Guide — базовая архитектура

### 9. Маркетплейс навыков / Определения навыков агентов

**Что:** Определение "навыков" агента как переносимых файлов в стиле SKILL.md, которыми можно делиться, версионировать и заменять. Навык "React Builder" против навыка "Python API Builder".
**Источник паттерна:** открытый стандарт Anthropic agentskills.io
