# Политика безопасности (Security Policy)

## Как сообщить об уязвимости

Если вы обнаружили уязвимость в системе безопасности Hermes Workspace, пожалуйста, сообщите об этом ответственно.

**НЕ открывайте публичный Issue на GitHub для сообщений об уязвимостях.**

Вместо этого сообщите об этом через [GitHub Security Advisories](https://github.com/outsourc-e/hermes-workspace/security/advisories) или в личные сообщения [@ericousodev в X](https://x.com/ericousodev).

Мы подтвердим получение вашего сообщения в течение 48 часов и постараемся предоставить исправление для критических проблем в течение 7 дней.

## Область применения (Scope)

- Код веб-приложения Hermes Workspace
- API-маршруты и взаимодействие с Claude
- Аутентификация и управление сессиями
- Обработка и рендеринг данных на стороне клиента
- Процесс одобрения выполнения команд (exec approval) и контроль "человек в цикле" (human-in-the-loop)
- Эндпоинты прокси браузера и скриншотов

## Вне области применения (Out of Scope)

- Сам Hermes Agent (сообщайте об этом в проект Hermes Agent)
- Сторонние зависимости (сообщайте соответствующим мейнтейнерам)
- Атаки с использованием социальной инженерии

## Меры безопасности (v3.0.0+)

**Аутентификация**

- Начиная с версии v3.0.0 все API-маршруты требуют аутентификации
- Токены сессий используют сравнение, устойчивое к атакам по времени (timing-safe comparison)
- Куки с флагами `httpOnly` + `SameSite=Strict`
- Аннулирование токена при выходе из системы

**Сеть**

- `Access-Control-Allow-Origin` ограничен только localhost — никаких подстановочных знаков (wildcard) CORS
- Эндпоинты прокси браузера и скриншотов заблокированы только для того же источника (same-origin)
- Ограничение частоты запросов (rate limiting) для высокорисковых эндпоинтов (доступ к файлам, отладка, выполнение команд)

**Данные и доступ к файлам**

- Предотвращение обхода путей (path traversal) на всех маршрутах файлов и памяти (`ensureWorkspacePath()`)
- Ограничение записи в память только файлами `.md`
- API-ключи и секреты никогда не передаются в клиентский код
- Токены Claude хранятся только на стороне сервера
- Диагностический вывод очищается от конфиденциальных данных

**Безопасность агента**

- Рабочий процесс одобрения выполнения (Exec approval) — конфиденциальные команды выполнения Claude требуют явного одобрения человеком через модальное окно в интерфейсе
- Сканирование безопасности навыков — каждый навык из маркетплейса проверяется на наличие подозрительных паттернов перед установкой

**Конфигурация**

- Файлы окружения добавлены в gitignore
- Эндпоинты конфигурации скрывают учетные данные в ответах
- Примеры конфигураций используют только плейсхолдеры ключей

## Пройденные аудиты безопасности

### SEC-3 (25.02.2026)

- Завершен полный аудит API на предмет покрытия аутентификацией; пробелов в защите приватных маршрутов не обнаружено (проверена аутентификация для `/api/config-get`, `/api/debug-analyze`, `/api/context-usage`, `/api/paths`).
- Добавлена принудительная проверка типа контента CSRF (`requireJsonContentType`) для оставшихся обработчиков POST, включая эндпоинты аутентификации и управления терминалом.
- Ужесточено ограничение частоты запросов до 10 запросов в минуту на IP (скользящее окно) для высокорисковых эндпоинтов:
  - `/api/terminal-input`
  - `/api/terminal-stream`
  - `/api/restart`
  - `/api/update-check` (POST)
