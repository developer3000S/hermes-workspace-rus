# Итерация 004 — Диалоги NPC без API (повышение автономности)

## Почему
Игрок сообщил о появлении "дампов" ошибок 401 от провайдера в чате NPC ("Error code: 401 ... token_invalidated"). Codex Plus и Anthropic OAuth достигли лимитов использования в середине сессии, из-за чего любые ошибки провайдера при свободном общении попадали напрямую в лог беседы. Цель: любое взаимодействие с NPC должно работать для любого посетителя БЕЗ API и БЕЗ авторизации.

## Что реализовано
- **Свободный чат теперь включается по желанию** через флаг окружения `VITE_PLAYGROUND_LLM_CHAT=1`. По умолчанию Playground работает полностью на заготовленных ответах (функционал уже реализован в предыдущих итерациях).
- **Обнаружение ошибок провайдера** в `/api/playground-npc`: теперь сервер сканирует контент ассистента на наличие паттернов ошибок 401 / token_invalidated / usage_limit / `{"error":...}` и заменяет их чистым ответом в стиле персонажа до того, как ответ покинет маршрут.
- **Клиент никогда не показывает "сырые" ошибки**: `playground-dialog.tsx` перехватывает все сбои и циклически переключается между 3 репликами-заглушками в стиле персонажа вместо вывода JSON или HTTP-кодов.
- **Индикатор "Скриптовый режим"** добавлен в заголовок диалога, когда LLM отключена или сработала заглушка, чтобы игроки знали, что мир намеренно работает в автономном режиме.
- **Лог беседы сохраняет образ персонажа**: реплики-заглушки специфичны для каждого NPC (`*Афина замолкает*`, `*Ирис наклоняет голову*`, `*Аполлон вздыхает*`), поэтому автономное состояние всё равно кажется естественным.

## Почему это правильный базовый вариант для хакатона
- Первому посетителю на `/playground` больше не нужно запускать Агента Гермеса, не нужен API-ключ или авторизация Codex.
- Квесты обучения Q1–Q5 уже продвигаются через заготовленные варианты диалогов (подтверждено в предыдущих итерациях).
- Если у посетителя *есть* работающий локальный шлюз Hermes и установлен флаг `VITE_PLAYGROUND_LLM_CHAT=1`, свободный чат включается автоматически и плавно переходит в скриптовый режим при любом сбое провайдера.

## Контекст провайдера (первопричина бага)
- Тариф Codex Plus достиг лимита использования в середине сессии ("You've hit your usage limit").
- Маршрут Anthropic OAuth уже выдавал ошибку `token_invalidated` от шлюза до этого.
- Шлюз возвращал **HTTP 200** с ошибкой вышестоящего сервиса, встроенной *внутри* контента сообщения ассистента, поэтому функция `openaiChat()` благополучно возвращала текст ошибки как ответ NPC.
- Фильтр `looksLikeProviderError()` в серверном маршруте Итерации 004 — это то, что предотвращает повторную утечку, даже если авторизация шлюза "слетит".
