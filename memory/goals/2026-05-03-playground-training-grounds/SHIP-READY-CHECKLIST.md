## 1. Как протестировать мультиплеер (5 мин)

### На одной машине (быстрый дымовой тест)
1. Жесткая перезагрузка Playground в текущей вкладке браузера: **Cmd+Shift+R**.
2. Откройте вторую вкладку → http://localhost:3005/playground
3. В обеих вкладках должно произойти следующее:
   - Отображаются имена друг друга над персонажами при перемещении
   - Индикатор количества игроков в HUD увеличивается до 2
   - Сообщения чата приходят друг другу (нажмите `Enter` в чате)
4. Присутствие в рамках одного браузера работает через BroadcastChannel + WS, так что это работает даже без сети.

### На разных устройствах (настоящий мультиплеер)
1. Телефон или вторая машина в **любой сети** (LAN или мобильная сеть).
2. На вашем рабочем Mac откройте порт 3005 — **самые простые варианты**:
   - `tailscale serve --bg http://localhost:3005` → используйте выведенный URL вида `https://aurora.tail***.ts.net`.
   - Или просто используйте публичный WS хаб напрямую: любой, кто запустит свою копию Playground, встретит вас в том же общем мире (WS хаб глобальный, сервер разработки локальный).
3. С тем файлом `.env`, который мы только что написали, оба клиента подключаются к:
   `wss://hermes-playground-ws.myaurora-agi.workers.dev/playground`
4. Проверьте в консоли браузера: вы должны увидеть одно WebSocket-соединение с этим URL и **отсутствие перехода на режим только BroadcastChannel** (индикатор транспорта показывает "ws" или "both").

### Проверка состояния хаба из CLI
```bash
curl https://hermes-playground-ws.myaurora-agi.workers.dev/health
# → {"ok":true,"online":N,"ts":...}

curl https://hermes-playground-ws.myaurora-agi.workers.dev/stats
# → {"online":N,"byWorld":{...},"peakToday":N,"peakDay":"..."}
```

- [ ] Тест вращения мышью: левая кнопка вращает, колесико зуммирует — всё должно работать быстро.
- [ ] Откройте 2-ю вкладку, чтобы подтвердить, что индикатор количества игроков загорелся.

## 5. Известные ограничения (упомянуть или скрыть)

- Ветка **не запушена** — тот, кто склонирует репозиторий сегодня, не увидит итерации 003–006.
- Интерьеры Forge/Grove/Oracle функциональны, но менее детализированы, чем Training/Agora.
- Текстуры земли — просто плоский цвет. Нормально для видео, немного скудно на скриншотах.
- Бесплатный тариф WS хаба на Cloudflare Workers — подходит для трафика хакатона, но не проверен под большой нагрузкой.
- Для диалогов через LLM требуется флаг `VITE_PLAYGROUND_LLM_CHAT=1` — по умолчанию используются скриптовые диалоги (намеренно).

## 6. Если что-то сломается во время записи

| Симптом | Решение |
| --- | --- |
| Камера застыла / левая кнопка не работает | Cmd+Shift+R; проверьте консоль на ошибки WS; эффект обзора мышью находится в `playground-world-3d.tsx` ~L939 |
| Имя другого игрока не отображается | Проверьте вкладку Network на предмет `wss://hermes-playground-ws.*/playground`; `.env` должен быть загружен Vite (перезапустите сервер, если нужно) |
| Нет звука | Сначала кликните в любом месте (браузеры блокируют звук до взаимодействия); проверьте кнопку Mute в панели утилит |
| Квест застрял | Попробуйте `localStorage.removeItem('playerProfile'); location.reload()` |
| WS хаб упал | Переключится на BroadcastChannel — мультиплеер в одном браузере всё равно будет работать |
| Forge Gate закрыт | Завершите все 5 шагов обучения; флаг `playerProfile.questProgress` управляет этим |
