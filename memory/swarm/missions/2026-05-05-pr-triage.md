# Миссия по триажу PR — 2026-05-05

**Репозиторий:** outsourc-e/hermes-workspace
**Цель:** провести триаж всех 19 открытых PR, подготовить структурированные отчеты по каждому, БЕЗ АВТОМАТИЧЕСКОГО СЛИЯНИЯ.
**Репортер:** Aurora (основная сессия)
**Время выполнения:** в рамках этой сессии (~30-45 мин для первых отчетов)

## Жесткие правила

1. **НЕ вливайте никакие PR.** Эрик должен лично одобрить каждое слияние.
2. **НЕ пуште в main или любую ветку контрибьютора.**
3. **НЕ закрывайте PR.** Просто классифицируйте их.
4. Отчитывайтесь перед Aurora с одним конкретным чекпоинтом по каждому PR.
5. Если столкнетесь с конфликтами, перечислите их; не разрешайте их в ветке PR.

## Результат по каждому PR

Для каждого назначенного PR подготовьте один структурированный блок:

```
PR #<num> — <заголовок>
АВТОР: <логин>
КЛАССИФИКАЦИЯ: APPROVE_READY | NEEDS_FIX | NEEDS_DISCUSSION | CLOSE_RECOMMEND
ОПИСАНИЕ ИЗМЕНЕНИЙ: <2-3 строки, что на самом деле меняется>
ЗАТРОНУТЫЕ ФАЙЛЫ: <количество + основные пути>
РИСК: low | medium | high — <причина в одну строку>
ЗАПУЩЕННЫЕ ТЕСТЫ: <pnpm build / pnpm lint / конкретные файлы тестов / N/A>
РЕЗУЛЬТАТ ТЕСТОВ: pass | fail | not-run — <доказательства>
КОНФЛИКТЫ: нет | <файлы>
БЛОКИРУЮЩИЕ ФАКТОРЫ: <или нет>
ЗАМЕТКИ РЕВЬЮЕРА: <2-4 строки — конкретные вещи, которые Эрик должен знать перед слиянием>
РЕКОМЕНДУЕМОЕ ДЕЙСТВИЕ: merge | request-changes:<что именно> | close:<почему> | discuss:<что именно обсудить>
```
